Game Features

-A dice that gets rolled to determine who goes first
-The ability to select a color
-Pawns that move around the board
-A game board with the appropriate sections and blocks
-A display for the current score

Game Screens

-This game has a defined style
-It has an interesting board, and sufficient texures on the pawns.
-The UI and information about the game is a bit limeted
-While it has a lot of style it lacks coherence.

-This version is very well connected.
-The UI and menus are informative
-The chat function on the game board is handy
-Style and complexity are a little on the low side

Design Patterns

There are two game elements for which I have used object design Patterns.
-The player
-The game
Both of these are implemented as different objects using the standard object constructor. The game constructor calls the player constructor. The feilds of the created object are later filled as players join and information is sent back from the clients

Plan of Action

-The dice must return a random number between 1 and 6
-Only one pawn is allowed to be selected at a time
-Each player has the choice of which color they want
-A player cannot choose an already chosen color
-There is a button to confirm the players choice of action
-There are mutliple different peices of text dynamically shown

CSS Inclusions

-Css animation in the background of the start screen.
-Css transition in the start screen menu buttons.
-Psuedo elements on both the start screen and the game screen.
-Fullscreen support on all pages.
-Media Queries to alert the player if his screen size is too small.
